# Android Roadmap

这份路线图基于两个前提：

1. `Xray-core-laochendeai` 继续负责 runtime 与桌面 / 本机控制面
2. `Xray-android-client` 负责 Android 客户端交付、交互和移动端运行控制

路线图优先做“用户能真实感受到的闭环”，而不是先把所有底层都铺满。

## 阶段 0：仓库和构建基线

状态：已完成

- 独立 Android 仓库
- 最小 app module
- 本地 debug APK 构建
- GitHub Actions debug APK artifact
- 基础 README / CLAUDE / 协作模板

退出标准：

- 新机器按文档可以打出 debug APK
- 仓库可以独立演进，不依赖继续塞回核心仓库

## 阶段 1：订阅导入与本地持久化

目标：让 Android 客户端先具备“进节点”的完整入口。

范围：

- 远程 URL 导入
- 手工粘贴导入
- 本地文件导入
- 本地订阅与节点数据持久化
- 基础 URI 解析与导入校验

建议补齐：

- 扫码导入订阅 / 节点
- 失败提示与导入错误可解释化

退出标准：

- 用户可以在手机上独立导入、查看、保存节点
- 删除订阅不会把节点历史粗暴抹掉

## 阶段 2：节点池与操作者主工作区

目标：把桌面端已验证过的节点池工作流搬到手机，但改成适合移动端的交互。

范围：

- 候选 / 验证中 / 活跃 / 隔离 / 已移除五池模型
- 手工加入验证、晋升活跃、移入隔离、恢复、彻底删除
- 节点情报卡片
- 节点干净度与网络类型展示

产品原则：

- 主工作区优先显示“能不能用、为什么不能用”
- 详情解释用抽屉、底部弹层或长按，不要默认铺满
- 诊断信息不抢主操作位

退出标准：

- 用户能在手机上完成节点筛选和池内操作
- 活跃池成为后续所有自动选择和运行控制的唯一候选集

## 阶段 3：目标站点绑定与策略路由

目标：把“哪个站点走哪个节点”做成手机可用的控制能力。

范围：

- 域名 / 站点绑定到指定节点
- ChatGPT / OpenAI 等常见目标的预设模板
- 白名单、直连、代理、策略组的移动端表达
- 绑定结果验证与命中反馈

退出标准：

- 用户可以在手机上明确指定某个域名走某个节点
- 命中错误时，界面能告诉用户“预期是谁、实际是谁”

## 阶段 4：核心集成与真实运行态

目标：从“管理客户端”进化到“可实际承载代理流量的 Android 客户端”。

范围：

- 选择核心集成方案：内嵌二进制、JNI、libXray 或其他封装
- 前台服务生命周期
- 基础运行日志与状态查看
- 配置生成、启动、停止、重载

高风险点：

- Android 版本兼容
- 电池优化 / 后台保活
- 前台服务与通知栏行为
- 运行时崩溃恢复

退出标准：

- Android 客户端可以在本机稳定启动和停止核心
- 节点选择与运行态切换形成闭环

## 阶段 5：移动端透明代理 / VPN Service

目标：真正进入手机侧代理承载，而不是只做控制台。

范围：

- VPN Service 接入
- Android 网络接管路径
- 分应用 / 分域名 / 分策略流量控制
- DNS 行为和旁路策略

说明：

这一阶段复杂度显著高于前几阶段，不宜在没有阶段 1 到 4 闭环前提前硬上。

退出标准：

- 能在真实 Android 设备上稳定运行
- 连接切换、恢复、停止后的系统状态可解释

## 阶段 6：可交付产品化

目标：从“能跑”走到“能发版、能维护、能给真实用户用”。

范围：

- 签名 release APK / AAB
- 版本检查与升级策略
- 崩溃采集
- 导入 / 导出 / 分享 / 二维码导出
- 文档、截图、发版流程

退出标准：

- 可以按版本节奏稳定发布
- release 包与 debug 包边界清楚

## 实施顺序建议

现实顺序建议是：

1. 先做阶段 1
2. 再做阶段 2
3. 再做阶段 3
4. 阶段 4 和阶段 5 分开推进，不要混在一个 issue

原因很直接：

- 没有订阅导入和节点池，手机端没有真实用户价值
- 没有目标站点绑定，移动端很难体现“为什么要指定节点”
- 运行态和 VPN Service 是高风险工程，应放到产品语义稳定之后

## 当前建议优先级

建议先开的 Android issue 主线：

1. 订阅导入与本地存储
2. 节点池数据模型与五池 UI
3. 节点情报卡片精简展示
4. 目标站点绑定与命中校验
5. 核心集成技术选型

## 非目标

当前不要把这些提前塞进第一阶段：

- 多节点聚合下载
- 通用多路径隧道聚合
- 付费、赞赏、商业化
- 一次性复刻桌面 WebPanel 全部页面
